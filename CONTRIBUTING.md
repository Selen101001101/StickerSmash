# CONTRIBUTING.md

![Commit Style](https://img.shields.io/badge/commit%20style-conventional%20commits-brightgreen)

## Welcome

Welcome to the **StickerSmash** project! We are excited to have you here. This repository serves as an Expo and React Native tutorial, guiding you through the process of building an application from scratch. Your contributions are invaluable, and we appreciate your interest in helping us improve this project.

## Getting Started

To set up your development environment, follow these steps:

### Prerequisites

- Node.js (version 14 or higher)
- Expo CLI
- Yarn or npm (package managers)

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/Selen101001101/StickerSmash.git
   cd StickerSmash
   ```

2. Install dependencies:
   ```bash
   npm install
   # or
   yarn install
   ```

3. Start the development server:
   ```bash
   npm start
   # or
   yarn start
   ```

## Branch Naming Convention

We follow a simple branch naming convention to keep our repository organized:

- **Feature branches**: `feature/<feature-name>`
- **Bugfix branches**: `bugfix/<bug-name>`
- **Hotfix branches**: `hotfix/<hotfix-name>`

For example:
- `feature/add-emoji-picker`
- `bugfix/fix-image-display`
- `hotfix/fix-navigation-issue`

## Commit Message Guidelines

We adhere to the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification for commit messages. This helps in generating a clear project history.

### Format

- Use the imperative mood in the subject line (e.g., "Add a new screen").
- Limit the subject line to 72 characters.
- Separate the subject from the body with a blank line.

### Examples

- `Add a pan gesture`
- `Create buttons using Pressable`
- `Update README.md`

## Development Workflow

1. **Fork the repository**: Click the "Fork" button on the top right corner of the repository page.
2. **Clone your fork**:
   ```bash
   git clone https://github.com/<your-username>/StickerSmash.git
   cd StickerSmash
   ```
3. **Create a feature branch**:
   ```bash
   git checkout -b feature/my-feature
   ```
4. **Make your changes** and commit them following the commit message guidelines.
5. **Push your changes**:
   ```bash
   git push origin feature/my-feature
   ```
6. **Open a pull request** against the `master` branch.

## Code Style and Standards

- Follow TypeScript best practices.
- Ensure your code is clean and well-documented.
- Write unit tests for new features and ensure existing tests pass.

## Pull Request Process

1. Ensure your pull request is against the `master` branch.
2. Provide a clear description of your changes.
3. Your pull request will be reviewed by the maintainers. Be open to feedback and make necessary adjustments.

## Issue Reporting

To report a bug or request a feature:

1. Navigate to the [Issues](https://github.com/Selen101001101/StickerSmash/issues) section of the repository.
2. Create a new issue with a descriptive title and detailed information.
